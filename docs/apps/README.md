# UtilityKit – App Catalogue

UtilityKit is an Expo / React Native (Expo Router) "all-in-one" mobile app that ships **~75 small utilities** under a single shell. Every tool lives at `app/(app)/tools/<tool>.tsx`, is registered in `constants/tools-meta.ts`, and is rendered as a card on the home grid (`app/(app)/index.tsx`).

This folder contains a per-category breakdown of every tool: **what it does, what's actually built, and what could be improved**. The goal is to give a single source of truth for product/engineering planning, refactoring, and roadmap discussions.

> Reference docs were generated by reading every tool source file directly. Where multiple tools share an architecture (e.g. the home-service trackers), it is called out explicitly.

## Tech baseline

| Concern | Stack |
|---|---|
| Framework | Expo SDK 54, React Native 0.81, React 19 |
| Routing | `expo-router` v6 with `(app)` and `(auth)` route groups |
| State / persistence | `@react-native-async-storage/async-storage` via `lib/storage.ts` (`KEYS` map) — every persistent tool reads/writes its own key |
| Theming | `components/ThemeProvider.tsx` + `constants/theme.ts` (Fonts, Radii, Spacing, light/dark colours) |
| Auth | `context/auth.tsx` (login/signup screens under `(auth)`) |
| Sensors | `expo-sensors` (compass, motion) |
| Crypto / secure storage | `expo-crypto` (Doc Vault PIN hashing, password gen randomness), `expo-secure-store` |
| Files | `expo-file-system`, `expo-document-picker`, `expo-image-picker`, `expo-sharing` |
| Animations | `react-native-reanimated`, `react-native-gesture-handler`, native `Animated` |
| Drag / sort | `react-native-draglist`, `react-native-sortables` |
| Fonts | `@expo-google-fonts/space-grotesk` |

## Conventions used in each tool doc

Each tool entry follows the same structure so they can be skim-read:

- **Purpose** — one-line summary.
- **Key features** — bullet list of what is *actually implemented* in source.
- **Tech notes** — persistence keys, APIs called, libraries, gotchas.
- **Improvements** — concrete, prioritisable suggestions (missing notifications, no export, weak validation, refactor opportunities, etc.).

A tool that has **no persistence** is a stateless calculator — anything entered is lost on close. This is flagged because it's frequently the single biggest UX gap.
